# Запуск бота с GitHub

GitHub хранит код, а сам Telegram-бот должен постоянно работать на хостинге.

## Вариант 1: Render

1. Откройте Render и подключите репозиторий:

```text
rsgsi77-code/-marketplace
```

2. Выберите Blueprint или Background Worker.
3. Если Render использует `render.yaml`, настройки подтянутся автоматически:

```text
Build Command: pip install -r requirements.txt
Start Command: python bot.py
```

4. Добавьте переменную окружения:

```text
BOT_TOKEN=токен_из_BotFather
```

5. Запустите сервис.

## Вариант 2: Railway

1. Создайте новый проект из GitHub-репозитория:

```text
rsgsi77-code/-marketplace
```

2. Railway должен увидеть Python-проект по `requirements.txt`.
3. Укажите команду запуска, если она не подтянулась автоматически:

```bash
python bot.py
```

4. Добавьте переменную:

```text
BOT_TOKEN=токен_из_BotFather
```

5. Запустите deploy.

## Важное про хранение данных

Сейчас бот хранит записи в локальном файле:

```text
state/entries.json
```

На некоторых облачных хостингах локальные файлы могут очищаться при перезапуске или новом деплое. Для теста этого достаточно. Для постоянной работы лучше следующим шагом подключить базу данных: PostgreSQL, SQLite на постоянном диске или Google Sheets.

## Проверка

После запуска откройте Telegram и напишите боту:

```text
/start
```

Потом добавьте тестовую запись:

```text
/quick WB; тестовый товар; сегодня; 2; 1000; 500; 15; 1.5; 100; 0; 200; 6; 0; 0; тест
```
